# Скрипты для мастер-класса

## Зачем скрипты

Вместо ручного копирования команд в терминал, мы сделали готовые bash-скрипты для каждого кейса и reset. Это позволяет:
- быстро запускать сценарии в отдельном терминале,
- не допускать опечаток,
- легко повторять кейсы,
- сфокусироваться на анализе метрик, а не на вводе команд.

## Почему bash

Bash — стандартный шелл во всех Linux/Mac средах, и он работает внутри контейнера или на хосте. Мы выбрали bash, потому что:
- удобно запускать из любого терминала;
- легко сохранять и реплицировать сценарии;
- не нужно устанавливать дополнительный инструментарий;
- можно запускать в фоновом терминале и одновременно наблюдать метрики в другом.

## Где скрипты

- `scripts/run_latency.sh`
- `scripts/run_error_rate.sh`
- `scripts/run_db_slow.sh`
- `scripts/run_memory_leak.sh`
- `scripts/run_chaos_mix.sh`
- `scripts/reset.sh`

## Как использовать

1. В одном терминале запускаешь сценарий:

```bash
bash scripts/run_latency.sh
```

2. В другом терминале наблюдаешь Grafana/Prometheus.
3. После проверки останавливаешь нагрузку (Ctrl+C) и сбрасываешь состояние:

```bash
bash scripts/reset.sh
```

Если нужно быстро переключиться по номеру кейса, можешь воспользоваться `scripts/run_case.sh`.

## Чтобы не ошибиться

- Запускай только один сценарий за раз.
- Всегда делай `bash scripts/reset.sh` после кейса, чтобы вернуться к baseline.
- Смотри метрики в Prometheus/Grafana до и после каждого запуска — это ключ.

## Что мы тестируем

- Latency: инъекция задержки, проверка p95/p99.
- Error rate: инъекция 5xx, проверка error rate.
- DB Slow: задержка БД, активные транзакции.
- Memory leak: накопление памяти, рестарты.
- Chaos Mix: комбинированные нагрузки.
