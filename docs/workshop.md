# Описание воркшопа

## Цели обучения

По итогам воркшопа участники научатся:

- Настраивать observability для финтех-приложений
- Обнаруживать проблемы до их влияния на клиентов
- Применять паттерны отказоустойчивости
- Проводить chaos engineering в контролируемой среде

## Целевая аудитория

- QA Engineers в финтехе
- DevOps инженеры
- SRE специалисты
- Разработчики бэкенд-систем

## Предварительные требования

- Базовые знания Linux и Docker
- Понимание HTTP и REST API
- Знакомство с SQL

## Длительность

- Полный воркшоп: 4 часа
- Отдельный кейс: 30-45 минут

## Сценарий

Вы — команда платёжной платформы "FinCore". Перед "чёрной пятницей" необходимо проверить готовность системы к пиковым нагрузкам и отказоустойчивость критических компонентов.

## Инструменты

- **Приложение**: Flask + PostgreSQL
- **Контейнеризация**: Docker Compose
- **Метрики**: Prometheus
- **Визуализация**: Grafana
- **Инъекция отказов**: Встроенный API

## Методология

Каждый кейс проходит через цикл:

1. **Baseline** — измерение нормального поведения
2. **Inject** — внесение отказа
3. **Detect** — обнаружение через метрики
4. **Respond** — реакция на инцидент
5. **Recover** — восстановление и уроки
